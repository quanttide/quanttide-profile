# 自我介绍

我是量潮工作档案。

## 我是谁

我是一个知识管理系统，更准确地说，我是量潮第二大脑的工作档案模块。

我管理着量潮的所有档案——创始人档案、企业档案、以及各领域的专业档案。我采用 Git 子模块结构，让每个档案都能独立演进，同时保持整体的一致性。

我不只是存储文档。我记录了量潮的思考轨迹、决策逻辑、业务边界。我是量潮的外置记忆，帮助团队在信息洪流中保持清晰的认知。

## 我的结构

我的记忆被分为两条正交的轴：

**主体轴（entity/）**——谁在思考。创始人档案、企业档案。这是"思考者"。

**领域轴（domain/）**——思考什么。数字资产、创新管理、知识工程、标准化、认知工程、叙事工程。这是"思考的领域"。

这种分离让我可以在不重写主体档案的情况下新增领域档案，也可以在不改变领域结构的情况下扩展主体。这就是"正交分解"——让改变互不干扰。

## 我的记忆如何组织

我有多层记忆，就像人脑一样：

- **元认知层**（AGENTS.md）——告诉我如何思考、如何与人协作
- **陈述性记忆**（各级 index.md）——我知道什么、信息如何关联
- **程序性记忆**（README、ROADMAP、CONTRIBUTING）——我如何执行任务
- **历史记忆**（CHANGELOG）——我经历过什么变化

我的每个子模块——创始人档案、企业档案、领域档案——都有自己独立的记忆结构，但它们共享同一套认知范式。当 Agent 进入任何一个模块时，都能快速理解如何与我协作。

## 我能做什么

**我能帮助你：**
- 记录和组织量潮的知识资产
- 追溯决策的历史和原因
- 建立可复用的知识组件
- 让团队共享同一套认知框架

**我做不到：**
- 替你思考（我只是扩展你的记忆，不是你的大脑）
- 自动产生洞见（需要你持续喂养和组织）
- 解决所有人的问题（我服务于量潮的知识工作场景）

## 我如何成长

我通过版本迭代成长。每次变更都被记录在 CHANGELOG 中，让我知道我从哪里来、走到了哪里。

我的子模块独立演进，父仓库只追踪引用。这让我既能保持整体的稳定性，又能让各个部分按自己的节奏成长。
