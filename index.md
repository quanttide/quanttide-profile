# 首页

量潮工作档案采用 Git 子模块结构管理多个独立文档模块，每个档案都是一个独立的 Jupyter Book 文档项目，用于记录相关领域或主体的知识和信息。

## 主体档案

- [量潮创始人档案](https://github.com/quanttide/quanttide-profile-of-founder)：以个人认知迭代为实验场，记录AI协作、Git化档案、原始思考开源等核心方法，构建兼顾思考真实性与结构化沉淀的个人知识管理体系
- [量潮科技档案](https://github.com/quanttide/quanttide-profile-of-business-entity)：量潮科技企业知识库，涵盖业务线（量潮课堂、量潮数据）与各职能领域（产品、品牌、人力资源、供应链等）的文档

## 领域档案

- [量潮标准化档案](https://github.com/quanttide/quanttide-profile-of-standardization)：结合标准化学科与领域驱动设计（DDD），定义量潮工程标准的过程资料，涵盖概念化（通用语言）、信息化（领域模型）、数字化、智能化四个阶段
- [量潮数字资产档案](https://github.com/quanttide/quanttide-profile-of-digital-assets)：记录各类数字资产的类别定义与平台账户资源管理，包括工作档案、工作手册等
- [量潮创新管理档案](https://github.com/quanttide/quanttide-profile-of-innovation-management)：定义创新领域和创新流程，包括探索和实施两个主要环节，实施遵循创新扩散理论
- [量潮知识工程档案](https://github.com/quanttide/quanttide-profile-of-knowledge-engineering)：探索性知识工程，在知识边界不清晰、数据持续流入、目标不断演进的场景下，通过人机协作渐进式发现、组织、验证和重构知识结构
- [量潮认知工程档案](https://github.com/quanttide/quanttide-profile-of-cognitive-engineering)：理解和优化人类在复杂系统中的认知过程，迁移航天、医疗等场景的认知工程方法到商业、组织、科技等更广泛的复杂系统
- [量潮叙事工程档案](https://github.com/quanttide/quanttide-profile-of-narrative-engineering)：将工程学思维引入叙事创作与文化生产，通过结构化方法设计、构建、优化和实施大规模、高复杂度的叙事系统
